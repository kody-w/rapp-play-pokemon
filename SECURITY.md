# Security Policy

## Supported versions

Security fixes are applied to the latest `1.x` release on `main`.

## Reporting

Please use GitHub's private vulnerability reporting for this repository. Do not
open a public issue containing a token, ROM, save, screenshot, recording,
runtime file, personal path, or exploit details. Revoke any credential that was
accidentally disclosed.

Include the affected commit, platform/Python version, impact, and minimal
ROM-free reproduction. Use generated bytes or mocks; never send copyrighted
game content.

## Security boundary

The setup and viewer servers are loopback-only and token-authenticated.
Mutating requests require an exact same-origin `Origin`; Host checks mitigate
DNS rebinding. Private runtime directories/files use `0700`/`0600`.

Only current gameplay screenshots and structured game state are sent to the
tool-free Copilot SDK session. ROM/save bytes, clips, tokens, logs, local paths,
and runtime files must never be model attachments or repository artifacts.

Do not port-forward or reverse-proxy either local server. Treat ROMs, browser
token URLs, save files, and recordings as private. Emulator, model, operating
system, and local-user compromise are outside this app's security boundary.
